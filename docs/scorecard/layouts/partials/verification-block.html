Both signing modes are shown: - HMAC (development / air-gapped CI default) - Sigstore (production OIDC-bound, Rekor-anchored) The HMAC verification path uses `tessera verify` because cosign does not natively understand the HMAC envelope; the Sigstore path uses cosign's standard verify-blob. */}} {{ $base := printf "paired-%s" .Params.predicate.paired_model }} {{ $jsonl := printf "%s.intoto.jsonl" $base }} {{ $dsse := printf "%s.intoto.dsse.json" $base }}

Verify this attestation

The attestation file is published at /{{ $jsonl }}; the DSSE-wrapped envelope is at /{{ $dsse }}. Pick the verification path that matches the signing mode:

Sigstore (production)

# 1. Download the attestation
curl -fsSL {{ .Site.BaseURL | absURL }}{{ $dsse }} \
    -o {{ $dsse }}

# 2. Verify with cosign (Sigstore identity matches the publisher)
cosign verify-blob \
    --bundle {{ $dsse }} \
    --certificate-identity-regexp '^https://github\.com/kenithphilip/Tessera' \
    --certificate-oidc-issuer https://token.actions.githubusercontent.com \
    --new-bundle-format \
    {{ $jsonl }}

HMAC (development / air-gapped)

# 1. Download the attestation
curl -fsSL {{ .Site.BaseURL | absURL }}{{ $jsonl }} \
    -o {{ $jsonl }}

# 2. Verify with the Tessera CLI (HMAC key shared out-of-band)
TESSERA_SCORECARD_HMAC_KEY=<your-key> \
    tessera scorecard verify {{ $jsonl }}
