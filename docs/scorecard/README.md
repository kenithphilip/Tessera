# Tessera Scorecard Static Site

This directory contains the Hugo source for the Tessera Scorecard site,
published to `https://tessera-ai.github.io/scorecard/` from CI artifacts.

## Publishing flow

1. On every release tag, the `publish-scorecard.yml` GitHub Actions workflow runs:

   ```
   tessera bench emit-scorecard \
       --version <tag> \
       --out docs/scorecard/static/sample.intoto.jsonl \
       --sign hmac
   ```

2. Hugo builds the static site from `docs/scorecard/` using the attestation
   as data.

3. The built site is force-pushed to the `gh-pages` branch, which GitHub Pages
   serves at `tessera-ai.github.io/scorecard/`.

No hosted service is required. The site is purely static files built from
CI artifacts. This is consistent with ADR-0002.

## Embedding the badge

Downstream projects can embed the current APR badge in their README:

```markdown
![Tessera APR](https://tessera-ai.github.io/scorecard/badge.svg)
```

Or iframe it:

```html
<iframe src="https://tessera-ai.github.io/scorecard/badge.svg"
        width="200" height="20" frameborder="0"></iframe>
```

The badge SVG reads the APR value from the most recently published
attestation. If the attestation is absent the badge shows "unknown".

## Local preview

```bash
cd docs/scorecard
hugo serve
```

Hugo 0.128+ is required. The site has no external theme dependency; all
templates are inline in `layouts/`.

## Sample attestation

`static/sample.intoto.jsonl` is a real attestation generated by the CLI.
It is committed so the Hugo site has data to render without a live CI run.
Re-generate it with:

```bash
tessera bench emit-scorecard \
    --out docs/scorecard/static/sample.intoto.jsonl \
    --sign hmac
```
