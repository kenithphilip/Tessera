# v1.0 API + ABI freeze

The v1.0 release pins the following surfaces for the v1.x line.
Breaking changes to anything below require a v2.0 major release
plus a 12-month deprecation window.

## Python API surface (not frozen)

These pieces remain stable but reserve the right to extend in
minor versions:

- `tessera.proxy` (FastAPI reference; production deployments use
  the Rust gateway or agentgateway)
- `tessera.adapters.*` (framework-specific adapters; minor
  surface may grow)
- `tessera.scanners.*` (new scanners may land in minor versions;
  existing scanners' signatures are frozen)
- `tessera.runtime.firecracker`, `tessera.runtime.tetragon` (Tier
  2 / Tier 3 wrappers; the underlying CLIs may evolve)
- `tessera.threat_intel.feed` (new IOC kinds may land in minor)

## Compliance taxonomies (frozen)

The 8 taxonomies Tessera enriches every event with:

1. NIST SP 800-53
2. CWE
3. OWASP Agentic Top 10 (ASI-01 through ASI-10)
4. MITRE ATLAS v5.4.0+
5. EU AI Act (Articles 9, 12, 14, 15)
6. ISO/IEC 42001:2023 Annex A
7. CSA AICM v1.0
8. NIST AI 600-1 (12 GenAI risks)

The mapping table itself may receive additive updates in minor
versions (new EventKinds get mapped; existing mappings do NOT
change).

## What can change in minor (v1.x) versions

- New `EventKind` values (additive)
- New scanners under `tessera.scanners.*`
- New compliance-taxonomy entries for new EventKinds
- New runtime tiers under `tessera.runtime.*`
- New optional `pip install tessera-mesh[...]` extras
- Bug fixes that change behavior in the direction of the documented contract
- Performance optimizations that do not change observable behavior

## What can change in patch (v1.0.x) versions

- Bug fixes
- Test additions
- Documentation
- Internal refactors with zero observable behavior change

## Forward compatibility commitments

| From | To | Guarantee |
| --- | --- | --- |
| v0.13 / v0.14 | v1.0 | All non-private callers continue to work; `min_trust` gets a `DeprecationWarning` but stays functional |
| v1.0 | v1.x | Above tables stay; minor additions only |
| v1.x | v2.0 | A 12-month deprecation window for any frozen surface that v2 changes; ADR-0007 scopes the planned ProvenanceLabel v2 changes |

## Verifying

A breakage of any frozen surface is detected by the v1.0 test
suite. Run:

```bash
.venv/bin/pytest tests/
cargo test --workspace --exclude tessera-py
```

A green run is the signal that the v1.0 freeze is intact.
